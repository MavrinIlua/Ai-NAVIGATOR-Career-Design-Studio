# Технический паспорт и архитектурная спецификация программного комплекса «AI Career Navigator Studio»

**Статус продукта:** Работающий прототип (MVP)  
**Область применения:** Цифровая профессиональная навигация, интерактивные профессиональные пробы в рамках предпрофессиональных классов и ИТ-направлений подготовки в вузе.

---

## 1. Назначение и функциональные возможности системы

Программный комплекс **«AI Career Navigator Studio»** представляет собой специализированное одностраничное веб-приложение (SPA), спроектированное для моделирования профессиональной деятельности и построения персонализированных образовательных траекторий в сфере искусственного интеллекта (ИИ) и машинного обучения (ML).

### Ключевые функциональные подсистемы:
1. **Подсистема динамического карьерного картирования (Career Explorer):**
   - Интерактивный каталог, содержащий классификатор из 20+ востребованных ИИ-профессий, сгруппированных по предметным областям (инженерные, инфраструктурные, аналитические, этические, продуктовые).
   - Движок динамической генерации карьерных паспортов на базе языковой модели `gemini-2.5-flash`, формирующий в реальном времени карту компетенций (Hard/Soft Skills), типичные должностные обязанности и пошаговый план обучения для выбранной специальности.
2. **Подсистема деятельностных проб и генерации медиаконтента (Image & Video Generator):**
   - Рабочее пространство для формулирования и тестирования текстовых запросов (промптов) к генеративным ИИ-моделям Imagen 3.0 и Veo 2.0.
   - Симуляция профессиональной деятельности ИИ-дизайнеров, промпт-инженеров и создателей медиаконтента (AI Creators).
3. **Подсистема локального ИИ-редактирования (Image Editor):**
   - Встроенный инструмент локальной модификации графического контента (Inpainting/Editing), позволяющий симулировать прикладные задачи постобработки и графической оптимизации в реальном времени.
4. **Рефлексивно-накопительная подсистема (Gallery):**
   - Локальная база данных выполненных работ ученика (сгенерированные паспорта профессий, иллюстрации, концепт-арты), формирующая цифровое портфолио прохождения профессиональных проб.

## 2. Технический стек и системные зависимости

Программное решение реализовано на базе современного веб-стека, оптимизированного для обеспечения высокой производительности и независимости от тяжелой серверной инфраструктуры на этапе развертывания в образовательных учреждениях:

* **Пользовательский интерфейс (UI):** React 19 (Functional Components, Hooks)
* **Язык программирования:** TypeScript 5.x (строгая типизация данных)
* **Сборщик проекта:** Vite 6.x (быстрая компиляция и минимизация бандла)
* **Библиотека стилей:** Tailwind CSS 4.x (утилитарный CSS для адаптивной верстки)
* **Иконки и графика:** Lucide React
* **Интеграция с ИИ-сервисами:** Официальный SDK `@google/genai` (версия ^1.38.0)
* **Анимации переходов:** Framer Motion (`motion/react`)

## 3. Архитектура клиентской части (Frontend Architecture)

Архитектурный каркас приложения построен по модульному принципу. Центральным источником состояния и координатором событий является компонент `App.tsx`.

### Схема взаимодействия компонентов:

```
                    ┌──────────────────────────────────────────┐
                    │                 App.tsx                  │
                    │   - Хранение глобального состояния       │
                    │   - Менеджер режимов (AppMode)           │
                    │   - Синхронизация с LocalStorage         │
                    └────────────────────┬─────────────────────┘
                                         │
         ┌───────────────────┬───────────┴───────────┬───────────────────┐
         ▼                   ▼                       ▼                   ▼
┌─────────────────┐ ┌─────────────────┐     ┌─────────────────┐ ┌─────────────────┐
│   Header.tsx    │ │CareerExplorer.ts│     │ImageGenerator.ts│ │   Gallery.tsx   │
│ - Навигация     │ │ - Выбор ролей   │     │ - Промпты       │ │ - Портфолио     │
│ - API Key Guard │ │ - Карта скиллов │     │ - Imagen 3 / Veo│ │ - Кэш работ     │
└─────────────────┘ └─────────────────┘     └─────────────────┘ └─────────────────┘
```

### Ключевые компоненты:
* `App.tsx`: Содержит корневое состояние приложения, включая выбранный режим работы (`mode`), флаг активности API-ключа (`hasApiKey`), массив сгенерированных элементов (`items`) и активный элемент в превью-баре (`latestItem`).
* `Header.tsx`: Верхняя панель управления, предоставляющая доступ к переключению модулей и интерфейсу безопасного ввода API-ключа.
* `CareerExplorer.tsx`: Модуль, отвечающий за отображение классификатора ИИ-профессий, отправку запросов к Gemini для формирования паспортов и визуализации дорожных карт развития.
* `ImageGenerator.tsx` и `ImageEditor.tsx`: Рабочие области генерации статического и динамического медиаконтента.
* `Gallery.tsx`: Интерактивная витрина сохраненных работ обучающегося с возможностью фильтрации по типам медиаданных.

## 4. Интеграция с Google GenAI API

Взаимодействие с нейросетевыми моделями осуществляется на клиентской стороне через изолированный сервис `geminiService.ts`, который обеспечивает ленивую инициализацию клиента Google GenAI SDK. Это предотвращает сбои на этапе статической сборки проекта при временном отсутствии переменных окружения.

### Схема инициализации клиента:

```typescript
import { GoogleGenAI } from '@google/genai';

let aiInstance: GoogleGenAI | null = null;

export function getGeminiClient(apiKey?: string): GoogleGenAI {
  if (!aiInstance) {
    // 1. Приоритет: Ключ, предоставленный пользователем в интерфейсе
    // 2. Альтернатива: Системная переменная окружения
    const key = apiKey || process.env.GEMINI_API_KEY || (window as any)._temporaryGeminiKey;
    
    if (!key) {
      throw new Error("Ключ доступа Google GenAI API отсутствует. Пожалуйста, укажите ключ в панели управления.");
    }
    aiInstance = new GoogleGenAI({ apiKey: key });
  }
  return aiInstance;
}
```

### Спецификация используемых моделей:
1. **`gemini-2.5-flash`** — основная модель для генерации текстового контента (профессиональные паспорта, учебные задания). Выбор обусловлен ультра-низким временем отклика (latency) и высокой точностью структурирования ответов.
2. **`imagen-3.0-generate-002`** — диффузионная модель нового поколения, используемая для генерации детализированных иллюстраций профессиональной среды по текстовым описаниям учащихся.
3. **`veo-2.0-generate-001`** — генеративная видеомодель, применяемая в модуле генерации медиа для симуляции создания анимированных презентаций и динамических видеороликов.

## 5. Механизм локальной персистентности данных (Data Persistence)

Для обеспечения автономной работы приложения в условиях школьных компьютерных классов со строгими сетевыми ограничениями и блокировками внешних БД, персистентность данных реализована на стороне клиента через API `localStorage` (ключ `nav_items_v3`).

### Оптимизация жизненного цикла Blob-ссылок:
Поскольку ссылки на бинарные объекты в памяти браузера (`blob:http/...`) инвалидируются при закрытии вкладки или перезапуске страницы, в конструктор начального состояния `App.tsx` встроен алгоритм автоматической валидации кэша:

```typescript
const savedData = localStorage.getItem('nav_items_v3');
const parsed = savedData ? JSON.parse(savedData) : [];

const validated = parsed.map((item: GeneratedMedia) => {
  // Если ссылка на видео ссылается на blob-объект из прошлой сессии,
  // она заменяется на пустую строку для предотвращения ошибок воспроизведения
  if (item.type === 'video' && item.url.startsWith('blob:')) {
    return { ...item, url: '' };
  }
  return item;
});
```

## 6. Дорожная карта научно-практического расширения комплекса

В рамках дальнейшего развития научно-исследовательской работы по профориентации предложен технический план реализации четырех функциональных расширений:

### Модуль А. Интеграция с API платформ поиска вакансий (например, hh.ru)
* **Задача:** Автоматический импорт реальных требований работодателей к ИИ-специалистам в реальном времени.
* **Техническое решение:** Развертывание легкого прокси-сервера (Proxy API) для обхода ограничений CORS, парсинг ключевых навыков из вакансий по тегам `key_skills` и их автоматическая группировка с помощью классификатора симулятора.

### Модуль Б. Модуль диагностического тестирования способностей
* **Задача:** Выявление предрасположенности обучающихся к различным группам ИИ-профессий (алгоритмическая, креативная, управленческая, этическая).
* **Техническое решение:** Внедрение интерактивной анкеты на базе психометрической модели, расчет баллов по категориям интересов и автоматическая передача вектора результатов в Gemini для формирования развернутого карьерного совета.

### Модуль В. Симулятор интерактивных практических кейсов (AI Sandbox)
* **Задача:** Проведение полноценных профессиональных проб (написание кода, аудит этики, проектирование моделей) с автоматической ИИ-оценкой.
* **Техническое решение:** Создание интерактивной песочницы, где написанный учеником промпт отправляется в Gemini для проведения автоматической оценки качества и устойчивости решения на основе жестких валидационных критериев.

### Модуль Г. Интерактивная карта образовательного маршрута (Roadmap)
* **Задача:** Визуализация индивидуального пути развития от школы до трудоустройства в виде интерактивного графа.
* **Техническое решение:** Использование библиотеки `reactflow` для построения блок-схем, где каждый блок (школьный курс, вузовская дисциплина, стажировка) генерируется ИИ под конкретного ученика на основе его интересов.
