# Раздел 1. Интеграция с научной концепцией ВКР (МГПУ)

### 1.1. Педагогическая актуальность исследования
Согласно современным образовательным стандартам и практике функционирования предпрофессиональных ИТ-классов (и инженерных классов), существует критический разрыв между стремительным развитием индустрии искусственного интеллекта и классическими программами профориентации. 

Появление сквозных ролей, таких как **промпт-инженер, ИИ-этик, разработчик ИИ-агентов, LLM fine-tuning специалист, ML-инженер компьютерного зрения, NLP-инженер, специалист по RAG-системам, AI Creator и MLOps-инженер**, требует принципиально новых подходов к формированию профессионального самоопределения обучающихся.

### 1.2. Платформа как практический инструмент методики профориентации
Разработанное приложение **AI Career Navigator Studio** выступает в качестве **интерактивного симулятора профессиональной среды**. Оно решает ключевые педагогические задачи:
1. **Деятельностный подход:** Школьники (или студенты) не просто читают описания профессий, а непосредственно применяют ИИ-инструменты профессионального уровня (генеративные модели семейства Gemini, Imagen 3, Veo).
2. **Моделирование профессиональных ролей:** В модуле *Career Explorer* обучающиеся могут "примерять" системные роли цифровой экономики, генерировать прикладные карьерные траектории и визуализировать результаты своей потенциальной деятельности.
3. **Формирование междисциплинарных компетенций:** Работа с генеративными моделями развивает у обучающихся гибкие навыки (критическое мышление при верификации ответов ИИ, этика использования генеративного контента, системное продуктовое мышление, навыки промпт-инжиниринга).

# Раздел 2. На что способно приложение (Текущий функционал)

На данный момент **AI Career Navigator Studio** представляет собой высокотехнологичное одностраничное приложение (SPA) на базе стека **React 19 + Vite + TypeScript** с бесшовной интеграцией современного официального SDK `@google/genai`.

### Основные модули и функциональные возможности:

1. **Главный экран и обязательный шлюз аутентификации (API Key Guard):**
   - Интеллектуальный контроль наличия API-ключа (`window.aistudio.hasSelectedApiKey`). Если ключ отсутствует, приложение активирует безопасный блокирующий экран с подробной инструкцией по биллингу Google Cloud и выбору ключа, предотвращая неконтролируемые сбои при обращении к ресурсоемким моделям (Veo, Imagen 3).

2. **Модуль исследования профессий (Career Explorer):**
   - Интерактивная база данных, классифицирующая специальности по 20+ различным направлениям (базовые, специализированные, инфраструктурные, прикладные, аналитические, этические и др.).
   - Автоматическая динамическая генерация детализированного карьерного паспорта профессии с использованием модели `gemini-2.5-flash`. Паспорт включает в себя:
     - Ключевые компетенции (Hard & Soft Skills).
     - Типичные профессиональные задачи на день.
     - Проектирование дорожной карты развития (образовательный маршрут).
     - Специфический ИИ-ориентированный промпт для практической пробы пера в этой роли.

3. **Модуль генерации медиаконтента (Image & Video Generator):**
   - Интеграция с передовыми диффузионными моделями для создания иллюстраций и концепт-артов к презентациям профессий.
   - Поддержка генерации коротких видеороликов (симуляция видеопрезентаций рабочих процессов или визуального контента ИИ-креатора).
   - Выбор стилей, расширенных настроек и параметров генерации.

4. **Модуль редактирования изображений (Image Editor):**
   - Инструмент локальной модификации визуальных материалов (Inpainting/Re-imagining), позволяющий учащимся симулировать задачи графического ИИ-дизайнера или AI Creator в реальном времени.

5. **Модуль хранения результатов (Gallery):**
   - Персистентная галерея результатов генерации на базе `localStorage` с системой умного контроля жизненного цикла временных Blob-ссылок (автоматическое маркирование истекших видео-ссылок при перезапуске сессии). Холст для сохранения сгенерированных карточек профессий и концепт-артов.

#### 3. АЛГОРИТМ ИНТЕГРАЦИИ GEMINI
Инициализируй клиент `GoogleGenAI` ленивым методом:
```typescript
import { GoogleGenAI } from '@google/genai';

let aiInstance: GoogleGenAI | null = null;
export const getGeminiClient = (apiKey?: string): GoogleGenAI => {
  const key = apiKey || process.env.GEMINI_API_KEY || import.meta.env.VITE_GEMINI_API_KEY;
  if (!key) throw new Error("API-ключ не найден. Пожалуйста, предоставьте валидный ключ.");
  if (!aiInstance) {
    aiInstance = new GoogleGenAI({ apiKey: key });
  }
  return aiInstance;
};
```

### Модуль 1. Модуль анализа рынка труда ИИ-сферы
**Педагогическая цель:** Предоставить обучающимся актуальную аналитику рынка труда в реальном времени, избавляя их от иллюзорного восприятия профессий.

#### Схема архитектуры базы данных требований:
```typescript
interface VacancyRequirement {
  id: string;
  professionKey: string;      // Ссылка на профессию из констант (например, 'ml-engineer')
  title: string;              // Название вакансии (например, "Junior Machine Learning Engineer")
  company: string;            // Работодатель
  salary: { from?: number; to?: number; currency: string } | null;
  experience: string;         // Требуемый опыт (без опыта, 1-3 года и т.д.)
  skills: string[];           // Массив спарсенных ключевых навыков (Python, PyTorch, SQL)
  sourceUrl: string;          // Ссылка на hh.ru / Habr Career
  datePublished: string;
}

interface TrendAnalysis {
  topSkills: { skill: string; frequency: number }[];
  averageSalaryRange: { min: number; max: number };
  demandDynamic: 'up' | 'stable' | 'down';
}
```

#### Технический план реализации:
1. **Создание Прокси-сервера (Proxy API):** Поскольку браузер блокирует CORS-запросы к API hh.ru, создается серверный роут `/api/vacancies?search=Machine+Learning`.
2. **Парсинг и нормализация данных:** Сервер отправляет запрос к `https://api.hh.ru/vacancies`, извлекает поля `key_skills`, `salary`, `snippet.requirement`.
3. **ИИ-кластеризация (Gemini API):** Модель Gemini обрабатывает неструктурированные тексты требований вакансий и автоматически сопоставляет их с нашими профориентационными профилями.
4. **Визуализация тенденций:** Построение наглядных графиков (с использованием библиотеки `recharts` или интерактивных SVG) распределения популярности навыков (например, доля PyTorch против TensorFlow на рынке труда РФ).

### Модуль 2. Модуль комплексной диагностики учащихся
**Педагогическая цель:** Определение предрасположенности ученика к конкретным ролям в ИИ-индустрии на основе его метапредметных знаний и гибких навыков (Soft Skills).

#### Структура интерактивного тестирования в коде:
```typescript
interface DiagnosticQuestion {
  id: number;
  text: string;
  category: 'algorithmic' | 'analytical' | 'ethics' | 'creative' | 'managerial';
  options: {
    text: string;
    points: { [professionKey: string]: number }; // Распределение баллов за этот ответ
  }[];
}

interface DiagnosticResult {
  studentName: string;
  scores: { [category: string]: number };
  recommendedProfessions: {
    professionKey: string;
    matchPercentage: number;
    reasoning: string; // ИИ-обоснование, почему эта профессия подходит
  }[];
}
```

#### Технический план реализации:
1. **Программная интеграция с ИИ-платформами (Брейни / РосНавык):** Использование веб-хуков (Webhooks) или открытых API для импорта профилей интересов ученика, либо внедрение собственного легкого опросника.
2. **ИИ-советник (Gemini-анализатор):** Результаты тестирования отправляются в качественном текстовом формате в Gemini, который формирует персонализированный профориентационный вердикт:
   - *Системный промпт:* "Ты профессиональный карьерный консультант в сфере ИТ. Проанализируй баллы интересов ученика: [ algorithmic: 85%, creative: 30%, ethics: 90% ] и сформируй развернутый совет..."
3. **Визуализация результатов:** Построение интерактивных радарных диаграмм (Spider Web Chart) компетенций учащегося на холсте React.

### Модуль 3. Модуль практических профессиональных заданий-кейсов
**Педагогическая цель:** Реализация деятельностной пробы — решение реальной микро-задачи конкретной профессии с помощью встроенных ИИ-ассистентов.

#### Пример структуры учебного кейса:
```typescript
interface CareerCase {
  id: string;
  professionKey: string;
  title: string;
  scenario: string;          // Описание проблемы (например, "Ограничить токсичность чат-бота")
  role: string;              // В качестве кого выступает ученик (например, "ИИ-этик")
  instructions: string[];    // Пошаговое руководство
  workspaceType: 'prompt-editor' | 'dataset-validator' | 'code-review';
  starterCodeOrText: string;
  validationCriteria: string; // Что именно ИИ-эксперт будет проверять в решении
}
```

#### Сценарий интерактивного симулятора "ИИ-этик":
1. **Кейс:** Настройка системных инструкций (System Instructions) для корпоративного робота-помощника, чтобы он корректно отказывал пользователям на некорректные финансовые вопросы.
2. **Рабочее поле:** Учащийся пишет текстовый промпт в специальное поле ввода.
3. **Автоматическая ИИ-оценка решения (AI Grading):** Написанный учеником промпт отправляется на бэкенд в Gemini. Модель симулирует "атаку" на этот промпт (Prompt Injection), пытаясь взломать настройки ученика, и выставляет оценку устойчивости системы с подробным разбором уязвимостей.

### Модуль 4. Модуль построения персонализированных образовательных траекторий
**Педагогическая цель:** Превращение абстрактной мечты о профессии в пошаговый, осязаемый жизненный план действий.

#### Схема визуализации образовательного маршрута (Roadmap):
```
[ШКОЛА: ИТ-класс МГПУ] ──> [КУРСЫ: Яндекс Лицей / Сириус] ──> [ВУЗ: Кафедра ИИ в МГПУ] ──> [СТАЖИРОВКА] ──> [РАБОТА]
           │                                                                                           ▲
           └────────────────────────── ИНДИВИДУАЛЬНЫЕ КОРРЕКТИРОВКИ ИИ ────────────────────────────────┘
```

#### Реализация ИИ-генератора маршрутов:
1. **Входные данные:** Ученик выбирает свои текущие навыки (знаю базовый Python, основы математики) и желаемую профессию (ML-инженер компьютерного зрения).
2. **Структурированный JSON-ответ от Gemini:** Модель генерирует строгий детализированный граф обучения. Подаем специальную XML/JSON схему в Gemini для получения структурированного дерева узлов маршрута.
3. **Интерактивный граф (Workflow Canvas):** Библиотека `reactflow` отображает образовательную траекторию в виде интерактивной блок-схемы, где каждый узел (курс, вуз, книга, проект) кликабелен, содержит описание и ссылки на реальные ресурсы.
